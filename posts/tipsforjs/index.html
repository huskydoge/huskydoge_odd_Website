Primitive and Reference Values

const只对primitive value生效，而对reference value无效！后者是栈上引用，堆上存储！但是想要完全改变一个const定义的对象，比如

const Object {
 		a:1,
 		b:2,
}

Object = {};

这样是不行的！改变property和改变object是完全两个概念！想要复制(Shallow Copy)Object可以用:

const objectCopied = Object.assign({},object)

Spread Operator

When we want the elements in the array individually, use spread operator

const arr = [7, 8, 9];
const newarr = [1, 2, ...arr];
console.log(newarr);

>>> [1, 2, 7, 8, 9]

但只有在原先是用逗号分隔的地方可以用哦！

也可以用作shallowCopy

const newArr = [...oldArr];

Optional Chaining

用于解决某个值或方法不存在的问题

// Values
console.log(restaurant.openingHours.Mon?.open);
//如果Mon不存在，那么不会访问open

// Methods
console.log(restaurant.order?.(0,1) ?? "Method doesn't exist");

// Arrays
const user = [{name:'John', email:'123@gmail.com'}];
console.log('user[0]?.name' ?? 'User array empty');

Immediately Invoked Function

// 这样的函数只能执行一次

(function (){
	console.log("This function will never run again");
})();

// 注意这里要加括号哦！

// Arrow Form
(()=> console.log("This function will never run again"));

Closures

(function () {
  const header = document.querySelector('h1');
  header.style.color = 'red';
  
  document.querySelector('body').addEventListener(
  'click', function () {header.style.color = 'blue';
                       }
  );
})();

这里， 变量header在Immediately Invoked Function里定义，当代码运行过这一段代码后，原先header就已经消失，而进入了document.querySelector('body').addEventListener中的function的闭包中。因此，这个函数在body对象被点击时，它仍能正常运行。

closure里的变量有很高的优先级。找变量会先找他。
