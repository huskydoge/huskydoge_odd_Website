14151. 【原4151】出栈序列

Description

现在有一个栈, 数字 1-n 按从小到大的顺序入栈. 给你一个出栈序列, 请你判断它是否合法.

Input Format

第一行为一个正整数 T, 表示数据组数;

接下来 T 行, 每行先是一个正整数 n, 紧接着 n 个数表示出栈序列.

Output Format

共 T 行, 每行一个字符串 Yes 或 No, 表示出栈序列是否合法.

Sample Input

2
3 1 3 2
3 3 1 2复制

Sample Output

Yes
No复制

Hint

T<=10, n<=10^6

时空磁盘限制（运行时）

时间限制： 4000 ms

内存空间限制： 512 MiB

磁盘空间限制： 无限制

Solution

分析

关键性质就一句话：对于数n右边的序列，所有小于n的数必须是逆序的。

这听起来有点绕，举个例子：

进栈序列：1 2 3 4

出栈序列： 1 4 3 2 
合法，因为 1 之后没有比它小的数；4之后比它小的数是 3 2，两个数字逆序排列；3之后比它小的只有一个2；2之后没有数字了。

出栈序列：1 4 2 3
非法，因为4之后比它小的数是 2 3，但是并非逆序排列。

要解释这种性质当然还是要从栈的FILO（First In Last Out）性质出发。

当一个数字在出栈序列里第一次出现的时候，凡是小于它的、还未出栈的数字，一定都压在它下方⬇️。还是用上面的例子

进栈序列：1 2 3 4
出栈序列：1 4 3 2

1首先出栈，而4是第二个出栈，在4出栈前，我们看到栈中的内容为（从上到下）：
- 4
- 3
- 2

或者说，想要让一个数N1出栈，我们必须先移走它上面的数字N2：

  • 如果N2存在，那么N2一定大于N1，所以导致N1进入出栈队列之前，N2已经进入出栈队列了
  • 如果N2不存在，那么N1可以直接出栈，这对应上面例子中数字 1 的情况

于是我们对关键性质做出了解释。数n右边的数就是还未出栈的数字，如果再加上小于性质，那么这些数就都压在n的下方。

代码

分析是一回事，但是怎么实现其实还有问题。

Idea1

一种简单的思路是：多重循环，对出栈序列a[n]进行检测。从出栈序列左侧向右移动，对每个位置i再从j = i + 1处开始扫描是否符合关键性质。

可以优化的地方是维护当前已经检查过的逆序数列第一个数字的最大值c，用于在第一轮扫描中进行判断，如果a[i]< c ，说明没有必要检查了，可以直接跳过。

因为c的意思是：当前i位置的右边数组中，所有小于 c 的数都是逆序排列的了。所以如果a[i]< c ，那么自然得出a[i]已经满足关键性质。

代码如下：


#include "iostream"
using namespace std;

inline void ans(bool flag){

    if(flag) cout << "Yes" << endl;
    else cout << "No" << endl;

}

int main(){
    int T = 0;
    ios::sync_with_stdio(false);//可关闭和stdio同步加速
    bool flag;
    long a[1000010]; // （0 base）
    long c = 1; // 用来记录 凡是小于 c的数都已经逆序，即不需要检查c之后的数了
    long tmp = 0;
    cin >> T;
    long n = 0;
    while(T > 0){
        flag = true;
        cin >> n;

        for(long i = 0; i < n; ++ i){
            cin >> a[i]; // write in numbers
        }

       for(long i = 0; i < n; ++ i){
           if(a[i] <= c) continue; // 在i的右侧，小于c的都已经是逆序了

           tmp = a[i];
           for (long j = i + 1; j < n; ++ j){
               if(a[j] < a[i]){
                   if(a[j] > tmp){
                       flag = false;
                       break;
                   } else {
                       tmp = a[j];
                   }
               }
           }
           if(!flag) break;
           else c = a[i];
       }

       c = 1;
        ans(flag);
        -- T;
}
}

时间复杂度:

  • 最好情况 O( $Tn$ )
  • 最坏情况 O( $T(n^2+Cn)$)

总体而言肯定是二阶的，于是很不幸，这样的写法会超时。

Idea2

另一种思路就是模拟法：我们来直接模拟这个出栈序列是怎么形成的。

我们首先关注一下出栈序列的第一个数，它是怎么来的？

答案是：它一进栈，就又出栈了，不带任何犹豫；而且前面的数都还没出栈。
