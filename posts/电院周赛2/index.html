括号消消乐

题目背景

祖玛这个游戏想必大家多多少少听说过。

现在让我们用括号序列来回忆一下祖玛这款童年经典吧。

题目描述

祖玛是一款消除类游戏，如果有若干个连续相同颜色的圆珠碰到一起就会消除。

在本题中我们对其做出简化。比如对于序列1221，相同的两个2会消除，接下来两个1碰到一起再次消除。为了数据传输更加简捷，这里我们使用括号序列来模拟。

规定如果某个序列中的一段可以消除，那么这一段序列的长度就是本次消除的得分。例如()()(())这段可消除序列，得分为8。（虽然形如()()这样的的序列可以算作两次得分，不过由于它们是连续的，因此我们也认为是一次消除。）

接下来，我们给出一段序列，希望你能求出这段序列中单次得分的最大值。

输入格式

一行字符串，保证其中仅含有前括号与后括号。

输出格式

一个正整数，表示该序列的最大单次得分值。

样例 #1

样例输入 #1

)()(())

样例输出 #1

6

样例 #2

样例输入 #2

))()())())

样例输出 #2

4

提示

对于25%的测试数据，字符串的长度不超过5000。

对于100%的测试数据，字符串的长度不超过5000000。

Solution

#include <iostream>
#include <stack>
#include <string>
using namespace std;
typedef long long ll;  

int main()
{
    string a;
    cin >> a;
    stack<int> s;
    ll ans = 0;
    ll i = 0, last = -1;
    while(i < a.length()) {
        if(a[i] == '(') s.push(i); //记录左括号下标
        else{
            if(s.empty()) last = i; //记录匹配失效位置
            else{
                s.pop();
                if(s.empty()){ //为空, 长度 = "当前位置" - "上一个匹配失效位置"
                    ans = max(ans, i-last);
                }
                else{ //非空, i-s.top() 计算这个右括号匹配到哪个位置
                    ans = max(ans, i-s.top());
                }
            }
        }
        ++i;
    }
    cout << ans << endl;
}

物资合并

题目背景

有物资的分配，自然也会有物资的合并。

题目描述

一听到我们又封控了，自然有很多的物资不断送到。显然，物资需要先整合后再统一分配以方便管理。现在有n堆物资围成一圈，每次合并只能合并相邻的两堆物资。假设每次合并物资时消耗的体力是要合并的两堆物资质量之和，合并后的物资重量是两堆物资加起来。通过若干次合并操作后，就能将这n堆物资合并为一堆了。

现在，爱偷懒的小Z想知道，合并物资所需要的最小体力消耗；而想通过这次物资合并来达到锻炼效果以减肥的小C想知道，合并物资所需要的最大体力消耗。

输入格式

数据的第 $1$ 行是正整数 $N$，表示有 $N$ 堆物资。

第 $2$ 行有 $N$ 个整数，第 $i$ 个整数 $a_i$ 表示第 $i$ 堆物资的重量。

输出格式

输出共 $2$ 行，第 $1$ 行为最小体力消耗，第 $2$ 行为最大体力消耗。

样例 #1

样例输入 #1

4
4 5 9 4

样例输出 #1

43
54

提示

对所有测试用例，

$1\leq N\leq 100$，$0\leq a_i\leq 20$。

Solution

这道题其实就是合并石子问题

  1. 如果只有一堆石子，不需要合并，费用为0；
  2. 如果有两堆石子，需要一次合并，费用为两堆石子数量之和；
  3. 如果有三堆石子，需要两次合并，费用为第1堆和第2堆合并以及第2堆和第3堆合并费用中的较小值，加上三堆石子数量总和；
  4. 如果有n堆石子，需要n-1次合并，可将这些石子分为第1到第k堆和第k+1到第n堆，k从1取到n-1，两部分分别求合并费用，k取使两部分的合并费用总和最小的那个值，最后再加上所有石子数量总和，即为整体的合并费用。

因此，状态转移方程为 $$ dp[i][j]=dp[i][k]+dp[k+1][j]+sum[i][j] $$ 其中，$dp[i][j]$表示将第i堆至第j堆的石子合并，$sum[i][j] $表示第 i 堆至第 j 堆石子数量总和。 其中，$i，j$ 分别表示第几堆石子，从第0堆石子个数为1，第1堆-2颗石子，第2堆-3颗石子，第3堆-4颗石子，第4堆-5颗石子。 当 $j-i=0$ 时，即为只有一堆石子，花费为0； 当 $j-i=1$ 时，即为有两堆石子； 当 $j-i=n$ 时，即为有n+1堆石子。

#include <cstdio>
#include <algorithm>
#include <cstring>
using namespace std;

const int M = 1e2 + 5;
int a[M];
int pre[M];
int f[M][M];

int main() {
	int n;
	scanf("%d", &n);
	memset(f, 0x3f, sizeof(f));
	for(int i = 1; i <= n; i ++) {
		scanf("%d", &a[i]);
		f[i][i] = 0;
		pre[i] = pre[i - 1] + a[i];//实现O(1)求sum(i,j)
	}
	for(int i = 2; i <= n; i ++) {
		for(int l = 1; l <= n - i + 1; l ++) {
			for(int k = l, r = l + i - 1; k < r; k ++) {
				f[l][r] = min(f[l][k] + f[k + 1][r] + pre[r] - pre[l - 1], f[l][r]);
			}
		}
	}
	printf("%d", f[1][n]);
	return 0;
}
