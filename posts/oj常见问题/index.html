C++ 编程中会犯的常见错误

HuskyDoge  included in OJ题解
 2022-11-30  2022-11-30  813 words   2 minutes    views  

初始化

变量或者对象的值没有初始化，或者类内某些成员没有初始化。数组没有初始化却被访 问。(可能引发SIGSEGV内存异常的中断)

所以，在初始化一个变量前，不要使用这个变量！（养成初始化变量的好习惯）

指针没有new就访问。此时指针值是随机值或者NULL，访问该位置的内存会引发 SIGSEGV中断。

  1. sort算法在algorithm库里。
  2. 看清楚数据规模，否则用long long 比较保险。typedef long long ll
  3. new了的内存最后最好要delete掉！

Delete

指针delete后再被访问或者再试图delete。此时指针值所在的内存位置已经没有对象， 任何访问都将引发SIGSEGV中断。

对象在生存期内，可能被多个指针共享。假若某个指针在某个地方删除了对象，然后别 的指针操作该对象时就可能出错。

指针常见错误🙅

指针没有初始化前的值是随机的，如果一旦初始化，它的值会发生变化！

指针未指向合法内存位置就访问

// 错误写法1, 没有为p申请空间，不可以传值！
int* p;
*p = 1;

// 错误写法2
int* p;
p = &1; // compiler error, 取地址运算符只能作用于左值(例如变量)

// 正确写法
int *p = new int;
*p = 1;
delete p; // 删除申请的空间
p = nullptr;

// 案例1
char* s;
strcpy(s, "SJTU"); // 错误, s 没有指向合法位置

指针回收相关

// 错误写法1
int* p; // p points to random position of memory 
delete p; // trouble

// 错误写法2 
void f() {
    int* p = new int[100];
    delete[] p;
// ... wait a while ...
delete[] p; // error, 不能回收一个已经被回收的内存
}

// 错误写法3
int* p1 = new int { 99 };
int* p2 = p1; // potential trouble
delete p1; 
// now p2 doesn’t point to a valid object p1 = nullptr; // gives a false sense of safety
char* p3 = new char { 'x' };
*p2 = 999; // error, 试图访问已被回收的内存 
cout << *p3;

// 错误写法4
// 指针 p 指向某个对象
delete p; // 回收该对象
if (p == nullptr) { // 错误, 指针p的值未必为空
  // ... something ...
}

指针的声明问题

Node* p1, p2, p3; // 注意p2, p3类型不是 Node*

指针赋值是地址值的传递

// 体会差别:

struct Node { 
   int data;
   Node* next;
};

Node* head = new Node; // method 1

Node* next = head->next; 
next = new Node;
//这里next和head->next的指针值起初是一样的了，后来又不一样了

// 与

// method 2

head->next = new Node; 
// 只有方法2可以正确的构造新节点。
